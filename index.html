Development

Most of my work is related to my job at the university.
I would like to highlight my favorite project saltwater. It is graphic engine developed in C++ and OpenGL 4.5 / OpenGLES 3.2. Saltwater is the engine framework to create and test new approaches inside a AR/MR context. In general, it is a little bit more then this. In fact it is possible to create little graphic applications with features like:

    Physically-based Rendering (PBR), HDR / Tone Mapping, Aliasing (FXAA, SMAA), Scripting, Editor, Augmented Reality (AR) using ARCore, Material editor, Asset browser, Caustics, and Refractions

Area Lights

Area lights are very important for a realistic and plausible rendering. Therefore, I've implemented area lights by using Heitz et al. Real-Time Polygonal-Light Shading with Linearly Transformed Cosines approach.

Augmented Reality

Augmented Reality (AR) is a very modern und upgrowing technology and one way to holograms. However, for now we have to solve several problems in AR like tracking and illumination. I try to reconstruct the environment lighting.

Reflective Shadow Maps

With the help of Reflective Shadow Maps (RSM) it is possible to create a realistic indirect lighting. This shot was made in combination with Screen Space Reflections.

Physically-based Shading

The shading is based on the article Moving Frostbite to Physically Based Rendering 2.0 by Sébastien Lagarde and Charles de Rousiers. I adopted some code parts for my saltwater engine.
